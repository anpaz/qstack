\title{qstack: A Kernel-Based Framework for Quantum-Classical Integration}

%\author{Andrés Paz}
%\email{anpaz@cs.washington.edu}
%\author{Dan Grossman}
%\email{djg@cs.washington.edu}


\begin{abstract}
Quantum programming faces unique challenges in integrating error correction, managing the quantum-classical boundary, and targeting diverse hardware platforms while maintaining program semantics. We present \textbf{qstack}, a compositional framework and language that inverts the traditional quantum-classical relationship through a stack-based execution model with measurement-driven callbacks. qstack's key innovations include hierarchical kernel composition, LIFO dynamic qubit allocation/deallocation, and a compilation infrastructure well-suited for quantum error correction. We provide a formal operational semantics that precisely defines program behavior. We demonstrate qstack's capabilities through several examples, including concatenated error correction and an implementation of the Steane code, and describe our implementation in Python featuring a state vector emulator, customizable noise models, and Jupyter integration. qstack represents a new approach to quantum programming models, ideally positioned for developing and testing fault-tolerant quantum algorithms across different abstraction levels and hardware architectures.
\end{abstract}
